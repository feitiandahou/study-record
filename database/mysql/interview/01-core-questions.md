# Core Interview Questions

## Index

- Why use B+ tree instead of a hash table or red-black tree?
- What is a covering index?
- Why can too many indexes hurt performance?

## Transaction

- What are ACID properties in practice?
- What is MVCC and what problem does it solve?
- What is the difference between redo log, undo log, and binlog?

## Lock

- What is the difference between row lock, gap lock, and next-key lock?
- How does a deadlock happen?
- How would you analyze deadlocks in production?

## Optimization

- What does `EXPLAIN` tell you?
- Why is deep pagination slow?
- How would you optimize a slow join query?

## Architecture

- How does master-replica replication work?
- What causes replication lag?
- When should you consider sharding?
