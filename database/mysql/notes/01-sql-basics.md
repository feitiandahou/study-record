# SQL Basics

## Scope

- select execution order
- where vs having
- inner join / left join
- group by and aggregate functions
- order by and limit
- subqueries and exists

## Must Know

- logical execution order is not the same as written order
- `where` filters rows before grouping
- `having` filters after grouping
- `left join` keeps rows from the left side even when no match exists

## Practice Targets

- write top N queries
- write grouped statistics queries
- compare `in` and `exists` in simple cases
- solve duplicate-data and pagination questions

## Interview Prompts

- What is the execution order of a select statement?
- What is the difference between where and having?
- When can a left join accidentally behave like an inner join?
